Accessing parameters

To get access to a specific QueryParam use the QueryParamGroup#get method by passing the name of the parameter to it. This will return the appropriate QueryParam or MultiQueryParam instance.

Remember that this refers to the name with which the parameter is associated in its group, not the URL parameter used to represent it on the URL.

Adding and removing parameters

Parameters can be added to or removed from a group using the QueryParamGroup#add and QueryParamGroup#remove, respectively. When a new parameter is added to a group and the group is bound this will resynchronize the entire group with the URL.

Reading the current value

The current value of a group can be accessed via QueryParamGroup#value. This returns an object with the parameter names as keys and the current parameter's value as values.

Observing value changes

The QueryParamGroup#valueChanges observable emits the current value of the group whenever it changes (see above for a description of the format of the emitted values). This observable is particularly useful in order to bind the group to some action such as querying your backend, see .

Always make sure that you unsubscribe from this observable if you no longer need the data. Note that it never completes on its own.

Changing the value

The value of a group can be changed programmatically through the patchValue and setValue methods. Similarly to their namesakes from the Reactive Forms API this performs either a partial or complete update of the value.
