In the previous article we learned how to configure our QueryParamGroup and wire it up with form controls. In this section, we take a look at how to work with the group programmatically.

QueryParamGroup

You can operate on the entire QueryParamGroup or access individual QueryParam instances to operate on.

Accessing individual parameters

To gain access to a specific QueryParam use the QueryParamGroup#get method by passing the name of the parameter model (i.e., not the param name for the URL) to it. This will return the appropriate QueryParam instance.

Reading the current value

The current value of the entire group is exposed in QueryParamGroup#value. This returns an object keyed by the names of the QueryParam models where each key carries the current value of the corresponding parameter.

Subscribing to value changes

In most cases you probably want to subscribe to changes in the value of the entire group in order to access some API or similar. This can be done by subscribing to the QueryParamGroup#valueChanges observable. It emits the current value as described above whenever the value of any parameter changes.

Remember to unsubscribe from this observable!

Updating the value

You can update the value of a QueryParamGroup either partially via patchValue or overwrite the entire value using setValue. Both methods expect the input to be of the same shape as QueryParamGroup#value as described above.

QueryParam

Working with an individual QueryParam is mostly the same as working with a QueryParamGroup.

Reading the current value

This is analogous to QueryParamGroup. You can find the API documentation under QueryParam#value.

Subscribing to value changes

This is analogous to QueryParamGroup. You can find the API documentation under QueryParam#valueChanges.

Updating the value

This is similar to changing the value of an entire QueryParamGroup, with the difference that QueryParam models support only the setValue method.
