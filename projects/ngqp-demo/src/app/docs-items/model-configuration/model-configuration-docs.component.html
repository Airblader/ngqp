The model defines which parameters you want to synchronize as well as how they are defined. A model is an instance of and is best created by using the service. In this article, we will explain the different features exposed by the model.

QueryParamGroup

serves as a container for the parameters. Having this container around your parameters is required as ngqp uses it to orchestrate its parameters as a unit. You can create a group by using the method.

By passing a second argument to , you can define additional settings which affect the entire group.

Controlling history state

When form control values change, ngqp synchronizes their values to the URL by navigating to the updated URL. By default, this will replace the current state in the history rather than appending to it such that it doesn't interfere with navigating back using, e.g., the back button.

By passing replaceUrl, this value can be overridden. Note that in ngqp this can be defined both globally (see ) and per , with the latter taking precedence over the former, if defined.

QueryParam

A model represents a specific query parameter in the URL. It contains defining information such as the name of the query parameter and how the form control values are (de-)serialized to and from the URL.

Query parameter name

ngqp separates the name of the parameter model from the name used in the URL because, for example, you may want to use ?q=â€¦ in the URL for a search input, but refer to it in your code with a more descriptive name such as searchText.

The param option of the parameter definition defines the parameter name used for the URL.

Converting between form control and parameter value

(De-)Serializers determine how the value of the form control is serialized into a string for the URL and, vice versa, how the parameter value is converted into the value used in the form control. For form controls using complex value types or for customization, you can use and pass your own (de-)serializer functions.

Parameters with multiple values

If your form control's model is array-typed, such as <select multiple>, you can set the multi option to true. This will use the default behavior of the router to have query parameters with multiple values.
