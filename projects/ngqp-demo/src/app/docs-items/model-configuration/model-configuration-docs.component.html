The model defines which parameters you want to synchronize as well as how they are defined. A model is an instance of QueryParamGroup and is best created by using the QueryParamBuilder service. In this article, we will explain the different features exposed by the model.

The QueryParamGroup

QueryParamGroup serves as a container for the parameters. Having this container around your parameters is required as ngqp uses it to orchestrate its parameters as a unit. You can create a group by using the method.

Group Settings

By passing a second argument to , you can define additional settings which affect the entire group.

replaceUrl

When form control values change, ngqp synchronizes their values to the URL using router navigation. This parameter mirrors the Router API and defines how ngqp should set this parameter on such navigations. If not set, ngqp falls back to the globally defined default (see ), which itself defaults to true, meaning that the history stack remains unchanged.
