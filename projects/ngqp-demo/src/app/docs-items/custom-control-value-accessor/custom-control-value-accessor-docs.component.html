Sometimes you may want to wire a parameter to a component which doesn't have a suitable ControlValueAccessor implementation. This article discusses different strategies on how to make such a use-case work.

However, before you use any of the strategies discussed here, you should consider providing a ControlValueAccessor implementation for your component. Remember that this is the mechanism that allows you to use the same form control with ngModel or reactive forms as well!

Strategy 1: Dynamic ControlValueAccessor

ControlValueAccessorDirective offers you a simple way to implement an on-the-fly control value accessor by wiring it up in the template directly.

Strategy 2: Manual wiring

If you really need or want to work without a ControlValueAccessor, you can use the API of QueryParam to wire it up manually. You can read more at , but specifically we will be using valueChanges and setValue.
